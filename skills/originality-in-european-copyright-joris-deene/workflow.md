# Workflow — Concrete originality analysis

> **Version 1.0** — Step-by-step procedure for applying the skill to a concrete object.

## Step 0 — Gather input

Required:

1. **The object** itself, or a precise description of it (photograph, text, technical drawing, code, …)
2. **The user's position**: pro-rightsholder or pro-alleged-infringer
3. **The factual context**: creation, author, circumstances, dispute
4. **Any reference objects**: state of the art, pre-existing work, alleged infringing object

If any of these elements is missing: actively request it before starting the analysis.

## Step 1 — Categorise the object

Identify the sector to which the object belongs:

- Text / press article / literary fragment / title
- Photograph / image
- Software / programming language / GUI
- Database / compilation
- Utilitarian article / design / applied art
- Audiovisual work / sporting event / live performance
- Other (taste, smell, performance — concept-of-work question)

This categorisation determines which sector-specific judgments primarily apply (see `methodology/`).

## Step 2 — Concept-of-work test (step 1 of the two-step test)

Question: is the object **sufficiently precisely and objectively identifiable**?

- Yes → proceed to Step 3
- Uncertain / no → strategic judgement:
  - Pro-rightsholder: establish that the object is objectively determinable
  - Pro-alleged-infringer: this is a strong line of attack — develop it fully
- Clearly no → analysis ends: no work within the meaning of EU law

## Step 3 — Originality test (step 2 of the two-step test)

Application of the Infopaq formula (paragraph 37) to the object.

### 3.1 Identify the creative choices

For each relevant aspect:

- Which **choices** were possible?
- Which did the author **make**?
- Which did the author **not** make?
- Were those choices **free** or were they **dictated** by external factors (technical, functional, conventional, rule-bound)?

### 3.2 Apply the "selection, arrangement, combination" test (Infopaq paragraph 45)

- Which elements are not protected as such? (Infopaq paragraph 46)
- Which **combination** or **arrangement** can carry the author's own intellectual creation?

### 3.3 Sector-specific refinement

Apply the relevant sector module to the concrete object.

## Step 4 — Develop the argumentation

Based on the chosen position:

- **Pro-rightsholder**: follow `positions/pro-rightsholder.md`. Build the positive argument (work + originality). Substantiate with the choices identified in Step 3.
- **Pro-alleged-infringer**: follow `positions/pro-alleged-infringer.md`. Choose one primary line of attack (concept of work OR originality) and build subsidiary attacks around it.

## Step 5 — Anticipate and rebut the counter-argument

An important step for a robust argumentation. Run through the anticipation matrix from the position file and build in rebuttals.

## Step 6 — Citation form

References to CJEU judgments:

> CJEU [date], C-[case number], *[short name]*, ECLI:[ECLI], paragraph X.

For example: CJEU 16 July 2009, C-5/08, *Infopaq International*, ECLI:EU:C:2009:465, paragraph 37.

## Step 7 — Output

Standard inline in chat. For formal documents: integrate with other relevant skills (procedural documents, legal opinions, memoranda).

## Concrete example workflow

> **Example (illustrative)**:
>
> *Client is the publisher of a press article; a competitor copies sentences of 10-15 words via a news aggregator.*
>
> **Step 1**: category → text / press article / short fragment
>
> **Step 2**: concept of work → the press article is plainly objectively identifiable; proceed
>
> **Step 3**: originality
>
> - 3.1: choices in form, presentation and language (Infopaq paragraph 44)
> - 3.2: elements (individual words) not protected, but their specific arrangement in a particular sentence may be
> - 3.3: Infopaq is the directly applicable case law here
>
> **Step 4** (pro-rightsholder): establish that the reproduced fragments themselves express the journalist's own intellectual creation (Infopaq paragraph 48), by reference to the specific word choice, formulation, and summary of the subject
>
> **Step 5**: anticipate "it is only a short fragment" → Infopaq paragraphs 47-48: qualitative test, no quantitative threshold
>
> **Step 6**: apply citation form correctly
>
> **Step 7**: deliver as advice or as the basis for a formal notice
