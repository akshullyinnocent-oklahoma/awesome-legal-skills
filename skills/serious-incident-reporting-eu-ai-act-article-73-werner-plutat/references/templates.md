## 2. Internal Escalation Checklist

### First 4 hours
- [ ] Confirm incident lead
- [ ] Confirm legal/compliance lead
- [ ] Preserve logs and deployment state
- [ ] Freeze version/build identifiers
- [ ] Confirm whether system is high-risk
- [ ] Check whether death / severe harm / rights impact / critical infrastructure is involved
- [ ] Identify Member State where incident occurred
- [ ] Determine likely deadline bucket

### First 24 hours
- [ ] Draft qualification memo
- [ ] Contact deployer/customer for missing facts
- [ ] Decide suspension / rollback / warning measures
- [ ] Prepare initial authority report
- [ ] Check GDPR / NIS2 / MDR / product safety overlap
- [ ] Prepare management summary

### First 7 days
- [ ] Complete root-cause workstream
- [ ] Decide field-wide corrective action
- [ ] Send supplementary report if needed
- [ ] Update instructions for use / customer guidance
- [ ] Document lessons learned in post-market monitoring
